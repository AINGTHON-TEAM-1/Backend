# Supabase PostgreSQL Best Practices Skill

This skill teaches the agent how to write high-quality, performant, and secure PostgreSQL code. It consists of approximately 30 rules categorized into 8 areas:

## Categories

### 1. [Advanced Features](./references/_sections.md#advanced-features)
- Optimization of JSONB, Full-Text Search, PostGIS, and other Postgres extensions.

### 2. [Connection Management](./references/_sections.md#connection-management)
- Guidelines for connection pooling, client lifecycles, and managing resource limits.

### 3. [Data Access Patterns](./references/_sections.md#data-access-patterns)
- Eliminating N+1 queries, using batch operations, and implementing pagination.

### 4. [Concurrency & Locking](./references/_sections.md#concurrency--locking)
- Handling transaction isolation, deadlocks, and lock contention.

### 5. [Monitoring & Diagnostics](./references/_sections.md#monitoring--diagnostics)
- Using pg_stat_statements, EXPLAIN ANALYZE, and performance metrics.

### 6. [Query Performance](./references/_sections.md#query-performance)
- Strategies for slow queries, missing indexes, and optimizing query plans.

### 7. [Schema Design](./references/_sections.md#schema-design)
- Guidance on table structure, data types, and indexing strategies.

### 8. [Security & RLS](./references/_sections.md#security--rls)
- Best practices for implementing Row-Level Security (RLS) policies and access control.

## Usage
Refer to specific rules in the `references/` directory when performing database-related tasks.
